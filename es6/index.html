ES6

Keith Gunn / @gunn_x

<talk>

  • 6th edition of ECMAScript
  • Also known as ES2015
  • Major update to syntax for writing complex applications
  • Not full browser support yet so being transpiled to ES5 (via Babel)

New variable scoping

  • Keywords: let and const
  • Original var had function scope only
  • let and const now have block scope
  • let is for variables that you plan to modify
  • const cannot be redeclared but does not mean immutable

New variable scoping


const foo = {"name": "Andy", "age": 34};
let bar = 10;

if (foo === 10) {
  let bar = 20;
}

foo.age = 35;
console.log(bar); // outputs 10
console.log(foo); // outputs {"name": "Andy", "age": 35}

Arrow functions

Improve writing of functions, especially callbacks

  • Concise syntax
  • Binds this automatically to current function
  • Implicit returns

Arrow functions


// ES5
var ages = [12,56,34,18,21,78];

var underAge = ages.filter(function(age) {
  if (age < 25) {
    return true;
  } else {
    return false;
  }
});
console.log(underAge);
          

// ES6
const ages = [12,56,34,18,21,78];

const underAge = ages.filter(age => age < 25);
console.log(underAge); // output [12,18,21]

Default arguments

You can now set default values for any argument


function multiplyBy(number, factor = 10) {
  return number * factor;
}

console.log(multiplyBy(5)); // outputs 50
console.log(multiplyBy(5,2)); // outputs 10

Template strings

Evaluation of variables inside strings


// ES5
var firstName = 'David';
var lastName = 'Bowie';
var fullName = "Hello my name is: " + firstName + " " + lastName;
            

// ES6
const fullName = `Hello my name is: ${firstName} ${lastName}`;
  • Uses character ` backtick
  • Supports multiline comments
  • Tagged templates also available, where you can run template string through a function first

New string methods

Few extra functions that help out, without need to resort to regex

  • .startsWith
  • .endsWith
  • .includes
  • .repeat

Destructuring

Allows us to extra data from Array/Objects into own variables

Objects


  const dog = {
    name: "Ralph",
    breed: "Border terrier",
    age: 2,
    sex: "M"
  }

  const {name, age} = dog;

  console.log(name, age); // outputs Ralph, 2

Destructuring

Arrays


const item = ['PS4 console', '02345644', 349.99];
const [name, sku, price] = item;

// or you might get data as CSV
const itemString = 'PS4 console,02345644,349.99';

const [name, sku, price] = itemString.split(',');

Destructuring

Can also destruct in function args


function renderName({title = 'Mr', first, last}) {
  return `${title} ${first} ${last}`;
}

console.log(renderName({first: "Bob", last: "Dylan"}));

Lots more you can do with destructuring!

Iterables

New for..of way to loop over iterables


const simpsons = ['Homer', 'Marge', 'Bart', 'Lisa', 'Maggie'];

// ES5
for (let i=0; i < simpsons.length; i++) {
  console.log(simpsons[i]);
}

// ES6
for (const simpson of simpson) {
  console.log(simpson);
}

For objects we need to stick with for..in until Object.entries/values is available in ES2017

Array upgrades

  • Array.from - create Array from array-like objects
  • Array.of - create Array from any number of arguments
  • Array.find - find first item in Array that matches callback fn
  • Array.findIndex - as .find except returns array index
  • Array.some - check if any item in array matches callback fn
  • Array.every - check if every item in array matches callback fn

Spread operator

Take every item from an iterable object and apply to parent


const animals = ['Elephant', 'Lion', 'Rhino', 'Giraffe'];

// copy array
const africanAnimals = [...animals];

// append arrays
const ukAnimals = ['Dog', 'Cat', 'Cow'];

const allAnimals = ukAnimals.concat(...africanAnimals);
console.log(allAnimals); // outputs ["Dog", "Cat", "Cow", "Elephant", "Lion", "Rhino", "Giraffe"]

// use spread to pass args
function calcBill(amount, tax, tip) { // }
var args = [20, 20, 10];
const total = calcBill(...args);

The spread can also work with Object but this in next version of JS

Rest operator

Basically the opposite of spread it combines items into array


// use in function arguments to handle any number
function suffixString(suffix, ...strings) {
  console.log(strings);
  return strings.map(string => `${string}${suffix}`);
}

const output = suffixString('ing', 'fly', 'wait', 'stir', 'stand');
console.log(output); // ["flying", "waiting", "stiring", "standing"]

// can also be used in destructuring
const data = ['ing', 'fly', 'wait', 'stir', 'stand'];
const [suffix, ...words] = data;

Promises

Promises have been around for a while but its now a standard

What is a promise? - it's a way to handle async and avoid callback hell!

A promise represents a value that might be available now, or in the future, or never

Can be chained together


const myPromise = fetch('http://yourserver/api/v2/articles');
// try to resolve promise
myPromise
  .then(data => data.json())
  .then(data => { console.log(data);})
  .catch(err => { console.log(err);})

Modules

There have been various attempts at modules for JS but now there is a standard

What is a module?

  • A way to share code in your application
  • It's a file with one or more functions
  • You can import functions from other modules
  • You can export function from your modules
  • Not supported in browser/node so must be transpiled

Modules - example


// doStuff.js
import slug from 'slug'; // default import
import { findKey } from 'lodash'; // named import

export const foo = 123;

function doStuff() {}

export default doStuff;

// app.js
import doStuff from './src/modules/doStuff'; // local file import

doStuff();
